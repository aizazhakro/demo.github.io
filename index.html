scv-demo-connector

Service Cloud Voice demo connector can help 3rd party telephony vendors provide a demo implementation of a connector. It will mock an incoming voice call after omni login, and also mock outbound calling and transfers.

Installation

Environment setup

The developer environment requires Node, NPM and webpack-dev-server.

Clone the repo

$ cd /directory/to/house/the/repo
$ git clone https://github.com/developerforce/scv-demo-connector

Install npm dependencies

$ npm install

Config Setup

Create a Public/Private key pair. This is required for Inbound calls. Add the public key into the demo.callCenter in reqTelephonyIntegrationCertificate field

Open config.env and set ORG_ID to the Pre Release Org Update ORG_NAME to your domain name in SCRT_URL Substitute the private key for PRIVATE_KEY. Use |NL| to separate lines

Launch application

$ npm start

By default the web server will run in SSL on port 8080.

Now that your web server is running, you can point your HVCC Call Center CTI Adapter URL to the web server url (i.e. https://localhost:8080)

Testing

Lint all the source code and run all the unit tests:

$ npm test

To bundle the source code in the src/ folder into one demo_connector.js file:

$ gulp bundle

To bundle the source code in the src/ folder into one minified demo_connector_min.js file:

$ gulp bundle --mode prod

DO NOT update lib folder or the scv-base-connector files

Local testing

For testing local changes, do the following:

Update the reqAdapterUrl in demo.callCenter to point to your localhost:8080 and create a contact center with this Run npm start. This will start the server for creating voice call records and launch the application Update the files as needed and refresh the browser page to get the new changes

Prerequisites

  • Create a Pre Release Org (228) https://www.salesforce.com/form/signup/prerelease-summer20/

  • Install the sample call center package via Workbench

  • Setup Omni: Enable perm OmniChannelEnabled, Go to Setup → Omni Channel → Enable Omni, Create a Service Channel of type 'PhoneCall', Create Presence Status 'Available' for the Phone channel, Go to the user profile of the agent (i.e. System Administrator) and under 'Enabled Service Presence Status Access' add the Available status for the profile. Lastly, add Omni to Utility Bar in App Manager.

  • Assign a user to this call center, and add the omni widget to the App you're using with Setup -> App Manager.

  • Assign users to the Permission Sets: Contact Center Admin (Service Cloud Voice BYOT User) Contact Center Agent (Service Cloud Voice BYOT User)

  • Purchase ServiceCloudVoiceBYOT License to enable Service Cloud Voice
